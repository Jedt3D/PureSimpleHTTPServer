URL Rewriting

PureSimpleHTTPServer supports URL rewriting and redirecting via a simple rule file. Rules apply to every GET request before the file is served.

Quick start

./PureSimpleHTTPServer --rewrite rewrite.conf

Rule syntax

rewrite  <pattern>  <destination>
redir    <pattern>  <destination>  [status-code]
VerbEffect
rewriteInternally rewrites the path — client sees nothing, a different file is served
redirSends an HTTP redirect response to the client (302 by default, or specify 301)

Lines starting with # are comments. Blank lines are ignored.

Pattern types

PrefixTypeExample
(none)Exact match/about — matches only /about
Trailing *Glob/blog/* — captures everything after the prefix
Leading ~Regex~/user/([0-9]+) — POSIX regex with capture groups

Destination placeholders

PlaceholderExpands to
{path}Text captured by * in a glob pattern
{file}Basename (last segment) of {path}
{dir}Directory portion (everything before the last /) of {path}
{re.1} … {re.9}Regex capture groups 1–9

Examples

Exact rewrite — clean URL

rewrite /about /about.html

Glob rewrite — blog slugs

# /blog/my-post  →  /blog/posts/my-post.html
rewrite /blog/* /blog/posts/{path}.html

Glob rewrite — preserve dir/file structure

# /static/img/logo.png  →  /assets/img/logo.png
rewrite /static/* /assets/{dir}/{file}

Regex rewrite — user profiles

# /user/42  →  /api/v1/users/42
rewrite ~/user/([0-9]+) /api/v1/users/{re.1}

Permanent redirect (301)

redir /old-page /new-page 301

Temporary redirect (302 — default)

redir /old /new

Per-directory rules

Place a rewrite.conf file inside any served directory. Rules in that file apply to requests whose URL path falls within that directory.

Evaluation order: global rules (from --rewrite FILE) are checked first; per-directory rules second. The first matching rule wins.

Per-directory rule files are cached and automatically reloaded when the file's modification time changes.

Integration with --clean-urls

Rewrite rules are applied before the clean-URL extension lookup. A rewrite to /page will then benefit from the .html extension fallback if /page does not exist but /page.html does.

ItemLimit
