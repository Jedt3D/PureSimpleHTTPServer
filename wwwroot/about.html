About PureSimpleHTTPServer

PureSimpleHTTPServer is a lightweight, zero-dependency static file server written entirely in PureBasic. It was designed to be the simplest possible tool for serving local files during development, running demo sites, or deploying small production workloads — with no runtime, no configuration sprawl, and a single binary you can drop anywhere.

Unlike Node.js or Python-based alternatives, PureSimpleHTTPServer compiles to a native executable for Windows, macOS, and Linux. The binary is self-contained: there is nothing to install, no package manager to run, and no background service to manage.

Key Features

  • Single native binary — drop it next to your wwwroot/ and run
  • URL rewriting and redirects via plain-text rewrite.conf files
  • Per-directory rewrite rules (place a rewrite.conf in any folder)
  • Automatic MIME type detection for 30+ file types
  • Pre-compressed .gz sidecar support (serve Brotli/Gzip without re-compressing)
  • Clean URLs — serve about.html at /about
  • SPA mode — fall through to index.html for client-side routing
  • Directory listings (opt-in)
  • Structured access logs (Apache Common Log Format)
  • Zero dependencies, cross-platform

Performance

30,000 req/s

< 3 ms latency

Median response time under load, measured with wrk on macOS Sequoia.

~1 MB binary

Compiled executable size. No runtime or framework to ship alongside it.

Built with PureBasic

PureBasic is a compiled, procedural language that produces small, fast native binaries. Its built-in network library gives direct access to socket I/O without any third-party runtime, making it an ideal choice for a lightweight server like this one.

The entire server — HTTP parsing, MIME detection, rewrite engine, and logging — fits comfortably in a few thousand lines of clean, readable PureBasic code.

Getting Started

Download the binary for your platform, place it alongside your website files, and run: 

# Serve the current directory on port 8080
PureSimpleHTTPServer --port 8080 --root ./wwwroot

Then open http://localhost:8080 in your browser. That's it.

For more details, see the Documentation or read the URL Rewriting guide.
