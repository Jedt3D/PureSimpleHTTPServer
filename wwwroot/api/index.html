API Explorer

These are sample JSON endpoints included with the PureSimpleHTTPServer demo site. They demonstrate how a rewrite.conf can route clean API paths to versioned JSON files on disk — no backend required.

Note: This directory uses a rewrite.conf to serve clean URLs like /api/v1/users without the .json extension. See the Rewrite rules section below for details.

Available endpoints

Users

GET /api/v1/users.json

Returns a list of 5 users with id, name, email, role, created_at, and active fields.

Products

GET /api/v1/products.json

Returns a list of 6 products with id, name, price, category, in_stock, and description fields.

Status

GET /api/v1/status.json

Returns server status, version string, uptime, request count, and active connections.

Fetching data with JavaScript

Because these are plain JSON files served with the correct Content-Type header, any standard fetch() call works without any special configuration: 

// Fetch the user list and log it to the console
fetch('/api/v1/users.json')
  .then(res => res.json())
  .then(users => {
    users.forEach(user => {
      console.log(`${user.name} <${user.email}> — ${user.role}`);
    });
  })
  .catch(err => console.error('Request failed:', err));

// Or using async/await
async function loadStatus() {
  const res = await fetch('/api/v1/status');   // clean URL via rewrite
  const data = await res.json();
  console.log(`Server v${data.version} — uptime ${data.uptime_seconds}s`);
}
loadStatus();

Rewrite rules

The rewrite.conf placed in the api/ directory maps each clean URL to its versioned .json file. This lets you publish stable public paths today and swap in a real backend (or a new API version) later without breaking existing clients: 

# api/rewrite.conf — make .json extension optional
rewrite /api/v1/users    /api/v1/users.json
rewrite /api/v1/products /api/v1/products.json
rewrite /api/v1/status   /api/v1/status.json

For a deeper explanation of the rewrite rule format, see the URL Rewriting guide.
